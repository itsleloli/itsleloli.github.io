Christmas Fight

3D FPS Splitscreen Game

Description

This project was made for a school assignment we had right before the Christmas break, where we had full creative freedom over the game and its gameplay except for three requirements:

  1. It had to have an overly Christmas theme.
  2. It had to be controllable with a controller.
  3. It had to be a split-screen multiplayer game.

On Tab (Input)

Most of the UI scripts work the same and are very similar to each other, so I'll quickly go over 1 example of what happens when you press Tab or the Tab input is received via the controller.

 
                
                    private void OnTab()
                    {
                        if (MainMenu.Instance.m_gameStarted)
                        {
                            if (m_pauseMenu != null && m_settingsMenu.activeSelf == false)
                            {
                                m_pauseMenu.SetActive(true);
                                m_ingameUI.SetActive(false);
                                MenuSelectBtn();
                                m_gamePaused = true;
                            }
                        }
                    }

It first checks a couple of things, whether the game is running and/or if the pause menu object actually exists, and afterwards it'll turn on the Pause menu and turn the In-game UI off. The MenuSelectBtn is for the controller input and highlights a standard button to round off. It sets the game Paused bool to true.



Audio Manager

The audio manager is a bit more sophisticated and is an Singleton. it works as followed during the initial setup it'll make an array of all sounds and put it in the variable m_sounds. a 

 
                    
                        //If this function is called it will play the audio clip with it's assigned string name.
                        public void Play(string clipname)
                        {
                            //Finds the audio Clip/Sound with the correct name and returns the sound as it is right now
                            Sound s = System.Array.Find(m_sounds, sound => sound.m_clipName == clipname);
                            if (s == null) return;
                            s.m_source.Play();
                        }

Then if a script wants to play a certain audio clip u can just call the Play function. then in the parameters u give the clipname and it'll look for a match through the array and play it if it exists. The same goes for a Stop function instead it looks for any playing audio clips and stops those.




Santa

The santa is responsible for dropping Presents down that have a variety of different weapons and he'll do so every minute at a random location through the map, after if he's reached his location he will despawn himself and respawn to a new location with a new package and a new location to drop. 

 
                     
                        private void LocationCheck()
                        {
                            //If Santa is at the desired drop distance start the SpawnPresent Coroutine.
                            if (Vector3.Distance(gameObject.transform.position, posToGoToo[randomGoToPos].transform.position) < m_randomDropDistance) m_spawnPresent = StartCoroutine(SpawnPresent());
                    
                            //If Santa has reached it's destination set the bool to true, start the Coroutine for spawning the next SantaClaus.
                            if (Vector3.Distance(gameObject.transform.position, posToGoToo[randomGoToPos].transform.position) < m_SantaMaxDistance)
                            {
                                santaReachedEnd = true;
                                m_spawnSanta = StartCoroutine(RespawnSantaClaus());
                            }
                        }

The script checks wether or not santa has reached his drop location yet and if so he'll start the SpawnPresent Coroutine which automatically drops a random present under him. if that's done he'll check wether or not he needs to despawn himself when he's gone beyond his designated position.




Reflection

I honestly had a lot of fun and learned a lot during this project because we had so much freedom and weren't bound to any ideas besides that we wanted it to be an FPS, so we were able to completely go crazy with the ideas for the mechanics and or gameplay for our game, hence I came up with the idea of adding random drops from a santa.

Looking back though I wish I would've put atleast some basic Post Processing in the game as well since, that would've made it just a bit more polished but at the time I sadly had almost zero knowledge about Post Processing. That being said I would like to come back and polish the game a bit more!

My favorite part of the game has to be the animations of the player, especially the "Crouch" where we just compressed the character model.
